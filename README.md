# 🔍 Interview Insight Search

Semantic search across customer interview transcripts — built as a demo for [Great Question](https://www.greatquestion.co).

## What it does

Customer researchers waste hours scrubbing through recordings to find key moments. This tool lets you search across all your interview transcripts semantically — type "pricing objections" and instantly surface every moment a customer mentioned price, even in different words.

- **Semantic search** using OpenAI `text-embedding-3-small` + FAISS vector index
- **AI insight summary** generated by GPT-4o across top results
- **Clean Streamlit UI** with example queries and relevance scoring
- Scales to thousands of hours of interview transcripts

## Demo

> Search: *"pricing concerns"*

The app surfaces every moment across 4 customer interviews where pricing came up — even when the exact word "pricing" wasn't used — and generates a synthesized insight summary across all results.

## Why I built this

Great Question's JD mentions *"semantic search across tens of thousands of interview hours"* as a core challenge. This is my take on that problem — a working prototype that shows how embeddings + vector search can turn hours of recordings into instantly searchable, insight-rich data.

## Stack

| Layer | Technology |
|-------|-----------|
| Embeddings | OpenAI `text-embedding-3-small` |
| Vector search | FAISS (IndexFlatIP with L2 normalization) |
| AI synthesis | GPT-4o |
| UI | Streamlit |
| Language | Python |

## Run it locally

**1. Clone the repo**
```bash
git clone https://github.com/venkatakrishna421/interview-insight-search.git
cd interview-insight-search
```

**2. Install dependencies**
```bash
pip install openai faiss-cpu streamlit tiktoken python-dotenv
```

**3. Add your OpenAI API key**
```bash
# Create a .env file
echo "OPENAI_API_KEY=sk-your-key-here" > .env
```

**4. Run**
```bash
streamlit run app.py
```

Open `http://localhost:8501` in your browser.

## How it works

```
Interview transcripts
        ↓
  Split into turns (speaker + timestamp + text)
        ↓
  Embed each turn → OpenAI text-embedding-3-small
        ↓
  Store in FAISS index (normalized for cosine similarity)
        ↓
  User query → embed → FAISS search → top-k results
        ↓
  GPT-4o synthesizes insight summary across results
        ↓
  Results displayed with speaker, timestamp, relevance score
```
